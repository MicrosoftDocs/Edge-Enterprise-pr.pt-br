# <span data-ttu-id="c4061-103">Criar variáveis de diretório de dados do usuário do Microsoft Edge</span><span class="sxs-lookup"><span data-stu-id="c4061-103">Create Microsoft Edge user data directory variables</span></span>

## <span data-ttu-id="c4061-106">Variáveis de caminho</span><span class="sxs-lookup"><span data-stu-id="c4061-106">Path variables</span></span>

<span data-ttu-id="c4061-107">Políticas para modificar caminhos de diretório de dados (por exemplo, configurando as variáveis de suporte [UserDataDir](microsoft-edge-policies.md#userdatadir) ou [DownloadDirectory](microsoft-edge-policies.md#downloaddirectory).</span><span class="sxs-lookup"><span data-stu-id="c4061-107">Policies for modifying data directory paths (For example, configuring the [UserDataDir](microsoft-edge-policies.md#userdatadir) or [DownloadDirectory](microsoft-edge-policies.md#downloaddirectory) support variables.</span></span> <span data-ttu-id="c4061-108">Ao configurar essas políticas, você pode usar variáveis em vez de caminhos embutidos no código.</span><span class="sxs-lookup"><span data-stu-id="c4061-108">When configuring these policies, you can use variables instead of hard-coded paths.</span></span> <span data-ttu-id="c4061-109">Por exemplo, para armazenar os dados de perfil em dados de aplicativo local do usuário no Windows, em vez do local padrão.</span><span class="sxs-lookup"><span data-stu-id="c4061-109">For example, to store your profile data under user local application data on Windows instead of the default location.</span></span> <span data-ttu-id="c4061-110">Definir a política [UserDataDir](microsoft-edge-policies.md#userdatadir) como **${local_app_data}\Edge\Profile**.</span><span class="sxs-lookup"><span data-stu-id="c4061-110">Set the [UserDataDir](microsoft-edge-policies.md#userdatadir) policy to **${local_app_data}\Edge\Profile**.</span></span> <span data-ttu-id="c4061-111">Na maioria das instalações de Windows 10, esse caminho resolve como *C:\Users\\&lt;Usuário atual&gt;\AppData\Local\Microsoft\Edge\Profile*.</span><span class="sxs-lookup"><span data-stu-id="c4061-111">On most Windows 10 installations, this path resolves to *C:\Users\\&lt;Current-user&gt;\AppData\Local\Microsoft\Edge\Profile*.</span></span>

>[!NOTE]
><span data-ttu-id="c4061-112">Se não houver definição, o Microsoft Edge usa os caminhos de diretório padrão.</span><span class="sxs-lookup"><span data-stu-id="c4061-112">If left unset, Microsoft Edge uses default directory paths.</span></span> <span data-ttu-id="c4061-113">Um usuário pode usar o sinalizador de linha de comando `--user-data-dir` para substituir o padrão.</span><span class="sxs-lookup"><span data-stu-id="c4061-113">A user can use the `--user-data-dir` command-line flag to override the default.</span></span>

### <span data-ttu-id="c4061-116">Diretrizes para usar variáveis de caminho</span><span class="sxs-lookup"><span data-stu-id="c4061-116">Guidance for using path variables</span></span>

<span data-ttu-id="c4061-117">Examine as diretrizes a seguir antes de usar variáveis de caminho.</span><span class="sxs-lookup"><span data-stu-id="c4061-117">Review the following guidance before using variables for paths.</span></span>

- <span data-ttu-id="c4061-118">Todas as políticas que envolvem caminhos em que Microsoft Edge armazena dados diferentes dependem da plataforma.</span><span class="sxs-lookup"><span data-stu-id="c4061-118">All policies that involve paths where Microsoft Edge stores different data are platform dependent.</span></span> <span data-ttu-id="c4061-119">Algumas dessas políticas estão disponíveis somente em plataformas específicas, mas outras podem ser usadas em todas as plataformas.</span><span class="sxs-lookup"><span data-stu-id="c4061-119">Some of these policies are available only on specific platforms, but others can be used on all platforms.</span></span>
- <span data-ttu-id="c4061-120">Para evitar erros causados por aplicativos a partir de locais diferentes em ocasiões diferentes, verifique se os caminhos são absolutos.</span><span class="sxs-lookup"><span data-stu-id="c4061-120">To avoid errors caused by applications starting from different locations on different occasions, make sure that paths are absolute.</span></span>
- <span data-ttu-id="c4061-121">Cada variável só pode ocorrer uma vez em um caminho.</span><span class="sxs-lookup"><span data-stu-id="c4061-121">Every variable can occur only once in a path.</span></span> <span data-ttu-id="c4061-122">Para a maioria deles, essa é a única maneira significativa de usar variáveis, pois elas são resolvidas como caminhos absolutos.</span><span class="sxs-lookup"><span data-stu-id="c4061-122">For most of them, this is the only meaningful way to use variables, because they resolve to absolute paths.</span></span>
- <span data-ttu-id="c4061-123">Quase todas as políticas criarão o caminho se não existirem (se possível nas circunstâncias existentes).</span><span class="sxs-lookup"><span data-stu-id="c4061-123">Almost all policies will create the path if it doesn't exist (if possible in the existing circumstances).</span></span>
- <span data-ttu-id="c4061-124">O uso de locais de rede para algumas políticas pode levar a resultados inesperados devido às diferenças em como diferentes versões/canais do Microsoft Edge tratam a estrutura de pastas.</span><span class="sxs-lookup"><span data-stu-id="c4061-124">Using network locations for some policies can lead to unexpected results due to differences in how different versions/channels of Microsoft Edge handle the folder structure.</span></span>
